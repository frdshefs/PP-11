# Система управления автосервисом — Интеграционное ядро (CSMS Integration Core)

Профессиональное интеграционное решение и бэкенд для автоматизации рабочих процессов автосервиса, конвейеров диагностического учета и транзакционной синхронизации между сервисами. Этот репозиторий содержит полную реляционную схему базы данных и функциональное консольное приложение для административного управления.

## 🚀 Ключевые архитектурные особенности
* **Паттерн Saga Orchestration:** Обеспечивает согласованность данных между компонентами (автомобиль, заказ, склад и уведомления) без использования жестких распределенных блокировок.
* **Гранулярная безопасность ACL:** Матрица модели прав доступа для нескольких ролей, охватывающая административное, операционное и техническое пространство через безопасные слои `GRANT`/`REVOKE` для ролей `administrator`, `receptionist` и `mechanic`.
* **Система аудита изменений (Audit Tracing):** Встроенные триггеры захвата измененных данных (CDC), отслеживающие модификации изменяемых состояний и записывающие их в виде структурированных JSON-пакетов в таблицу `audit_logs`.
* **Механизм мягкого удаления (Soft-Purge):** Реализация безопасного управления жизненным циклом сущностей с помощью флагов изоляции (`is_deleted` / `is_active`), что позволяет избегать физического удаления данных и нарушения реляционных связей.

## 🛠️ Топология базы данных и схема сущностей

Слой хранения данных состоит из **12 строго типизированных сущностей**, именованных в нижнем регистре на английском языке:

1. `roles` — классификация уровней доступа в системе.
2. `users` — реестр идентификации для аутентификации.
3. `clients` — домен мастер-данных клиентов с поддержкой флага `is_deleted`.
4. `employees` — справочник персонала со структурным отслеживанием должностей и ограничений по `salary`.
5. `vehicles` — метаданные автомобилей клиентов, привязанные к логам пробега `current_mileage` и записям `vin`.
6. `bays` — планировщик физических ресурсов ремонтных боксов мастерской.
7. `orders` — основной транзакционный документ, выступающий якорем для конечных автоматов (state machines).
8. `order_statuses_history` — последовательный, доступный только для чтения журнал жизненного цикла статусов.
9. `parts` — складская книга инвентаря, управляющая критическими лимитами запасов запчастей.
10. `order_parts` — промежуточная связь «многие ко многим», фиксирующая динамические цены для накладных.
11. `notification_logs` — асинхронный трассировщик уведомлений (шаблон интеграционного Outbox).
12. `audit_logs` — глобальный неизменяемый регистратор системных операций и аудита.

## 🏁 Руководство по быстрому запуску и развертыванию

Выполните следующие последовательные шаги, чтобы скомпилировать, инициализировать и запустить программное решение на своем рабочем месте «из коробки».

### Шаг 1: Инициализация базы данных MS SQL Server

1. Запустите **SQL Server Management Studio (SSMS)** или используемые вами инструменты CLI.
2. Подключитесь к вашему экземпляру базы данных (например, Имя сервера / Server Name: `F` или именованный локальный экземпляр).
3. Откройте новое окно запроса (New Query), скопируйте все содержимое из файла `database/database_setup.sql` и выполните его (**F5**).
4. Убедитесь, что в выводе написано `Commands completed successfully` и что созданная база данных `carservice_db` содержит все 12 таблиц, представления (views) и процедуры.

### Шаг 2: Настройка строки подключения в проекте C#

1. Откройте директорию с исходным кодом проекта в **Visual Studio** или **VS Code**.
2. Перейдите к файлу `src/CarServiceManagement/Program.cs`.
3. Найдите определение переменной `connectionString` в верхней части файла:
```csharp
private static string connectionString = "Server=F;Database=carservice_db;Trusted_Connection=True;TrustServerCertificate=True;";

```


4. Если ваша конфигурация SQL Server использует именованный экземпляр (например, `F\SQLEXPRESS`), измените параметры инициализации соответствующим образом, используя экранированный строковый литерал (с символом `@`):
```csharp
private static string connectionString = @"Server=F\SQLEXPRESS;Database=carservice_db;Trusted_Connection=True;TrustServerCertificate=True;";

```



### Шаг 3: Установка необходимых зависимостей NuGet

Откройте консоль диспетчера пакетов (Package Manager Console) вашей IDE или встроенный терминал, перейдите в папку проекта и убедитесь, что библиотека официального драйвера SQL-клиента успешно подключена к ссылкам сборки:

* **Через терминал .NET CLI:**
```bash
dotnet add package Microsoft.Data.SqlClient

```


* **Через графический интерфейс NuGet в Visual Studio:**
Найдите `Microsoft.Data.SqlClient` во вкладке **Обзор** (Browse) и нажмите **Установить** (Install).

### Шаг 4: Запуск приложения

Нажмите **F5** внутри Visual Studio или запустите приложение из командной строки с помощью инструментария компилятора .NET:

```bash
dotnet run --project src/CarServiceManagement/CarServiceManagement.csproj

```

После этого откроется окно терминала с текстовым иерархическим интерфейсом панели управления автосервисом.

## 📊 Включенные аналитические представления и операционные процедуры

Система исключает сложные вычисления на стороне клиента, перенося критически важные интеграционные и аналитические метрики непосредственно на уровень высокопроизводительных скомпилированных конструкций SQL:

* `view_active_repairs` — обеспечивает структурную отчетность по текущим рабочим нагрузкам в ремонтных боксах.
* `view_low_stock_parts` — оценивает складские запасы и генерирует предупреждения, когда остатки опускаются ниже установленных лимитов.
* `view_mechanic_performance` — рассчитывает ключевые показатели эффективности (KPI) персонала на основе объемов выручки.
* `sp_create_draft_order` / `sp_add_part_to_order` / `sp_complete_repair_saga` — атомарные транзакционные эндпоинты, обернутые в надежные блоки исключений `TRY...CATCH`, которые автоматически выполняют стратегию отката (rollback) в случае сбоя бизнес-логики.
